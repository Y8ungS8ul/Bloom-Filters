Курсовой проект: «Оценка эффективности реализации фильтра Блума в практических задачах: сравнительный анализ с альтернативными структурами данных»

Автор: Максим Костин, ИВБ-215, ПГУПС.

--------------------------------------

1. Запуск программы.

Программа начинает запуск с консоли, в которой пользователю
предлагается выбрать алгоритм.

1 - BF - выбор Фильтра Блума и всех возможных его модификаций
2 - HLL - консольная реализация алгоритма подсчета уникальных
элементов HyperLogLog

2. "1 - BF"

При выборе фильтра Блума пользователю предложено посомтреть на
некоторые реализации:

	1 - Фильтр Блума [Классика]
	2 - Фильтр Блума с подсчетом
	3 - Инверсивный фильтр Блума
	4 - Сравнение Фильтра Блума с Y-fast trie
	5 - Cuckoo-фильтр

3. В программе также предусмотрен GUI , построенный с помощью библиотеки SFML

Для перехода необходимо выбрать: 

	1-BF => 1 - Фильтр Блума [Классика] => 3 - интерфейс SFML

4. Для демонстрации работы многопоточной обработки файла в 500000 строк необходимо выбрать:

	1-BF => 1 - Фильтр Блума [Классика] => 2 - threads

5. Графический интерфейс приложения.

При выборе открытия интерфейса на SFML пользователь получит соответствующее.
окно с возможностью выбора структуры данных для визуализации и просмотра диаграм.

Основные элементы управления:

[1] - начать запись из файла таблицы (.csv/.db) в 1000/10000/100000 строк
[2] - Иная визуализация схожей структуры / выбор первой диаграммы сравнения
[3] - Иная визуализация схожей структуры / выбор второй диаграммы сравнения

Для выхода в предыдущие пункты меню требуется зажимать 'ESC'.

Чтобы вернуться из подпунктов меню в главное меню (начало программы, первая страница)
необходимо нажимать "Left"-стрелку на клавиатуре ("<-").

Для добавления данных и демонстрации работы простейших вариаций структур данных
следует нажимать "A", "B", "Q".
